# Internationalization (i18n)

## Setup

Using `next-intl` with App Router.

### Supported Locales

Defined in `src/i18n/routing.ts`:

- `en` (English) - default
- `es` (Spanish)

## Translation Files

Location: `src/i18n/messages/`

```json
// en.json
{
  "common": {
    "loading": "Loading...",
    "save": "Save"
  },
  "auth": {
    "login": "Log in"
  }
}
```

## Usage

### In Components

```tsx
import { useTranslations } from "next-intl";

export function MyComponent() {
  const t = useTranslations("common");

  return <p>{t("loading")}</p>;
}
```

### Nested Keys

```tsx
const t = useTranslations("landing");
t("hero.title"); // Accesses landing.hero.title
```

## Navigation

Use i18n-aware navigation:

```tsx
import { Link, useRouter } from "@/i18n/navigation";

// Links automatically include locale
<Link href="/dashboard">Dashboard</Link>;

// Router
const router = useRouter();
router.push("/login");
```

## Adding a New Locale

1. Add to `src/i18n/routing.ts`:

```typescript
locales: ['en', 'es', 'fr'], // Add 'fr'
```

2. Create `src/i18n/messages/fr.json`

3. Add translations

## Middleware

Location: `src/middleware.ts`

Handles:

- Locale detection from browser
- Locale prefix in URLs
- Protected route checking
